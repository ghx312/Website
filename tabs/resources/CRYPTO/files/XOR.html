Vulnerabilities

Key Reuse Vulnerability:

This can reveal the plaintext through statistical analysis or a known-plaintext attack

Known-plaintext Attack:

Knowing part of the plaintext can allow you to know part of the key, increasing vulnerability for future attacks

Bit flipping attack: (Linear Separability)

By flipping the bits in a ciphertext, you can manipulate the message as XOR has a very linear system, where any change in any variable will cause a change in other variables

Poor Randomness of Key:

If the key generated is not truly random, statistical analysis can be used to crack the key and hence the message from the ciphertext

Single-Byte XOR Vulnerability:

Frequency analysis can be used to find out which byte it is.

Frequency Analysis:
XOR is not safe from frequency analysis when the key is reused, as XOR has TERRIBLE diffusion properties, making the plaintext susceptible to frequency analysis

TLS 1.0 CBC Mode:

This is a cryptographic protocol that can be decrypted by exploiting XOR’s vulnerabilities, by using chosen-plaintext attacks and BEAST attacks, by exploiting the XOR and IVs and message data. (IV: Pseudorandom value used at the start of an encryption program)

Advanced Obfuscation Attack:

Using statistical analysis and differential analysis, detect repeated samples or weak points
